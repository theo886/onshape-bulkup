PURPOSE
-------
Assigns upload levels to CAD files based on their dependency relationships to determine optimal upload order for migrating from SolidWorks PDM to Onshape.

KEY FEATURES
------------
✓ Reads Excel upload list and PDM references CSV
✓ Calculates hierarchical upload levels (0=non-CAD, 1=parts, 2+=assemblies)
✓ Uses recursive algorithm with memoization for efficiency
✓ Handles circular dependencies gracefully
✓ Adds 6 new columns to Excel file
✓ Provides detailed statistics and orphan detection
✓ Supports dry-run mode for preview
✓ Follows existing codebase patterns

ALGORITHM
---------
1. Parse references.csv into dependency graph: Map<assembly, [children]>
2. For each file:
   - .SLDPRT files → Level 1
   - Non-CAD files → Level 0
   - .SLDASM files → Level = max(child assembly levels) + 1
3. Use memoization cache to avoid recalculating
4. Detect and handle circular dependencies

NEW COLUMNS ADDED
-----------------
1. uploadLevel      - Integer (0, 1, 2, 3, ...)
2. zipPath          - Empty string (user fills in)
3. uploadStatus     - "pending" (initial value)
4. onshape:documentId - Empty string (filled during upload)
5. onshape:elementId  - Empty string (filled during upload)
6. folder           - Empty string (user fills in)

COMMAND LINE OPTIONS
--------------------
-i <path>     Input Excel file (default: Upload/Onshape_Upload_List.xlsx)
-r <path>     References CSV (default: PDM/references.csv)
-o <path>     Output file (default: overwrites input)
--dry-run     Preview without writing
-h, --help    Show help

INTEGRATION WITH WORKFLOW
-------------------------
This script is designed to be run before the upload process:

1. Export references from PDM → references.csv
2. Prepare upload list → Upload List.xlsx
3. Run assignLevels.js → Add levels and columns
4. Use levels to upload in correct order:
   - Level 0: Non-CAD files
   - Level 1: Parts
   - Level 2+: Assemblies (bottom-up)
