## What's Working

### NEW: Unified Upload Tool
Single Excel-driven script that handles the entire migration:

```bash
# Assign levels to files based on dependencies
node assignLevels.js -i Upload/Onshape_Upload_List.xlsx -o Upload/Onshape_Upload_Levels.xlsx

# Upload everything (processes in level order)
node unifiedUpload.js -i Upload/Onshape_Upload_List.xlsx

# Or upload by level for testing
node unifiedUpload.js --level 1  # Parts only
node unifiedUpload.js --level 2  # Assemblies only
```

### Key Features
- **Excel-driven**: All file info, properties, folders in one spreadsheet
- **Level-based processing**: 0=non-CAD, 1=parts, 2+=assemblies (depth-sorted)
- **Auto-relink**: Assemblies automatically relink to master parts
- **JSON sidecar**: Crash-safe status tracking (upload_status.json)
- **Document grouping**: Files with same `document:name` go to same doc
- **Folder handling**: Creates folders if they don't exist

### New Files Created
| File | Purpose |
|------|---------|
| `assignLevels.js` | Assigns uploadLevel based on assembly dependencies |
| `unifiedUpload.js` | Main migration script - Excel-driven |
| `lib/relink.js` | Reusable relink module for assemblies |
| `Upload/Onshape_Upload_Levels.xlsx` | Output with levels assigned |

### Core Scripts (Still Available)
1. `uploadStandaloneParts.js` - Upload master parts as composite Part Studios
2. `uploadAssemblies.js` - Upload Pack & Go ZIPs
3. `relinkAssemblies.js` - Full relink with grouping and origin fastening

### Key Implementation Details
- `createComposite: true` - All parts import as composite (handles multi-body)
- `yAxisIsUp: true` - Correct orientation for both parts and assemblies
- `includeNonSolids: true` - Required to detect composite parts in assembly API
- `includePartTypes: ['PARTS', 'COMPOSITE_PARTS']` - Insert both part types
- Group + Fasten to Origin - Locks assembly in place
